# Os Passos do Hacking

Independente do hack que você quer criar — seja um aimbot elaborado ou simplesmente infinito de dinheiro — o processo segue sempre os mesmos quatro passos:

1. **Identificar** o que você quer mudar.
2. **Entender** que tipo de memória você precisa encontrar.
3. **Localizar** essa memória no jogo em execução.
4. **Alterar** essa memória.

Simples assim. A complexidade varia de hack para hack, mas a receita é sempre essa.

# Passo 1: Identificar

Antes de tudo, você precisa saber exatamente **o que** quer modificar no jogo.

Parece óbvio, mas é importante ser preciso. "Quero mais dinheiro" é diferente de "quero ver jogadores através das paredes":

* Modificar o dinheiro do jogador → mudança de uma **variável de memória**
* Ver através das paredes → mudança no **código que desenha as paredes**

O que você quer modificar determina completamente a abordagem que você vai usar.

# Passo 2: Entender

Agora que você sabe o que quer, precisa entender **onde** está essa informação na memória.

Existem três tipos principais de modificação:

* **Variáveis** — valores simples como dinheiro, vida, munição
* **Código** — as instruções que controlam o comportamento do jogo (ex: como as paredes são desenhadas)
* **Arquivos** — dados salvos em disco, como inventário ou save game

Entender com qual tipo você está lidando define quais ferramentas e técnicas você vai precisar.

# Passo 3: Localizar

Com o alvo definido e o tipo de modificação claro, você parte para a **caça**.

Para variáveis, a ferramenta mais usada é um **scanner de memória** — como o [Cheat Engine](https://www.cheatengine.org). Ele vasculha a RAM do jogo procurando o valor que você quer encontrar.

Para código, você precisa de um **debugger** (depurador) — como o [x64dbg](https://x64dbg.com) ou o [Ghidra](https://ghidra-sre.org). Com ele, você pode pausar o jogo, inspecionar as instruções sendo executadas e encontrar exatamente o ponto que quer modificar.

Depending on the game and its protections, this step can range from minutes to hours. Games protected by modern anti-cheats like EAC or BattlEye add extra layers of difficulty here.

> **Dica:** Jogos sem anti-cheat (open source, single-player, ou legados) são os melhores para aprender. É por isso que usaremos o **Wesnoth** nas primeiras aulas — ele é gratuito, de código aberto e não tem proteção anti-cheat alguma.

# Passo 4: Alterar

Você encontrou a memória. Agora é hora de **mudar**.

No começo, você vai fazer isso diretamente pelas ferramentas (Cheat Engine, x64dbg). Depois de confirmar que funciona, você pode escrever um programa para fazer isso automaticamente — aí você tem um hack de verdade.

Esse fluxo (encontrar manualmente → confirmar → automatizar) é o ciclo central de desenvolvimento de qualquer hack.

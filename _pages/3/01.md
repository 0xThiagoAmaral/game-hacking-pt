# Visão Geral

Nas aulas anteriores, aprendemos a hackear o Wesnoth usando um scanner de memória e um debugger. Essas mudanças funcionavam — mas apenas enquanto o jogo estava aberto. Ao fechar e reabrir, teríamos que refazer tudo do zero.

Além disso, seria impossível distribuir essas modificações para outras pessoas.

A solução? **Programar o hack**. Criamos um programa que automaticamente encontra e modifica a memória que queremos sempre que é executado. Uma vez programado, você pode rodá-lo sempre que quiser — e compartilhar com quem quiser.

# Linguagens de Programação

Como mencionamos na aula de Fundamentos de Computação, existem várias linguagens de programação. Para game hacking, precisamos de uma linguagem que:

- **Acesse a API do Windows diretamente** — para abrir processos e manipular memória
- **Modifique a memória de outros processos** — usando funções como `ReadProcessMemory` e `WriteProcessMemory`
- **Execute diretamente no hardware** (linguagem compilada) — para ter controle total e máxima performance

Essas exigências eliminam linguagens interpretadas (como Python puro) e linguagens sem acesso direto à memória (como Java). Em 2026, as opções mais viáveis são:

| Linguagem | Uso em Game Hacking |
|---|---|
| **C++** | ⭐ Principal escolha — controle total de memória, compatível com a API do Windows |
| **C#** | Boa alternativa — mais fácil que C++, funciona via P/Invoke para acessar a API do Windows |
| **Rust** | Crescendo em popularidade — segurança de memória, mas mais difícil de integrar com Windows |
| **Python** | Útil para automação e scripting, via ctypes ou bibliotecas como `pymem` |

Para este curso, usaremos **C++** — a linguagem que oferece o melhor equilíbrio entre potência e controle para game hacking.

# C++

C++ é uma linguagem compilada. Você escreve código legível, o compilador transforma em opcodes, e o resultado é um `.exe` que roda direto no processador.

Para game hacking, a feature mais importante do C++ é a capacidade de criar **ponteiros** — variáveis que armazenam endereços de memória. Com ponteiros, você pode ler e escrever em qualquer endereço da RAM.

# Ponteiros

Ponteiros são uma das features mais poderosas — e mais intimidadoras — do C++. Mas a ideia é simples:

> **Metáfora:** Um ponteiro é como um **post-it com um endereço escrito**. Em vez de guardar o dado em si, você guarda onde o dado está guardado.

Exemplo:
```c++
int x = 5;
int *y = &x;    // y guarda o endereço de x
```

- `int *y` → declara `y` como um ponteiro para inteiro
- `&x` → o operador `&` retorna o **endereço** de `x`

Agora, se fizermos:
```c++
*y = 6;
```

O operador `*` "segue" o ponteiro — vai até o endereço que `y` aponta e muda o valor lá. Resultado: `x` também vira 6.

Aplicando a game hacking: se você encontrou o ouro do jogador em `0x12345678` e esse endereço é estático:

```c++
int *gold = (int*)0x12345678;    // ponteiro para o endereço do ouro
*gold = 999;                     // altera o valor diretamente na memória
```

Ao executar, o ouro em `0x12345678` vira 999.

# Tipos de Hacks Programáveis

Há três formas principais de programar um hack:

| Tipo | Como funciona | Quando usar |
|---|---|---|
| **Executável externo** | Programa independente que usa API do Windows para ler/escrever a memória de outro processo | Hacks simples de memória, fáceis de distribuir |
| **DLL injetada** | Biblioteca carregada dentro do processo do jogo; acessa a memória diretamente via ponteiros | Hacks complexos que precisam reagir a eventos do jogo (teclas, etc.) |
| **Wrapper customizado** | Substitui uma DLL do jogo (ex: DirectX/OpenGL) por uma versão modificada | Hacks visuais como wallhack, chams |

Nas próximas aulas, vamos construir os dois primeiros tipos. Começaremos com o executável externo.
